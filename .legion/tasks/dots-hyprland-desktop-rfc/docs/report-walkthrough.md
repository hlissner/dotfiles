## Reviewer Summary

This implementation completes the no-downgrade continuation requested by the task contract: Axiom now targets the imported end4 `ii` Quickshell desktop path instead of the old Axiom shell or a substrate-only service layer. The imported source and Nix wiring are documented in `docs/import-manifest.md`; validation and review are recorded in `docs/test-report.md` and `docs/review-change.md`.

The key reviewer question is whether the repository now contains and wires the end4 runtime source while keeping Nix in charge of host facts, services, generated-state boundaries, and rollback. Existing evidence says yes for repository-local readiness, with live Wayland/Hyprland behavior still requiring validation in the actual graphical Axiom session.

## What Changed

- Imported upstream end4 `dots/.config/quickshell/ii` at commit `bebf66da89cd2afa4738da47fb3a0a9fa5af7035`, including `shell.qml`, modules, services, panel families, scripts, assets, defaults, translations, and the rounded-polygon submodule dependency.
- Imported adjacent runtime sources used by the `ii` path: `config/matugen`, `config/fuzzel/fuzzel.ini`, `config/hypr/hyprland.conf`, `config/hypr/hyprland/**`, `config/hypr/hypridle.conf`, and `config/hypr/hyprlock.conf` plus lock helper scripts.
- Changed Quickshell ownership so `modules/desktop/quickshell.nix` defaults to `configName = "ii"` and links `quickshell/ii`, `matugen`, and `fuzzel` into XDG config.
- Demoted the legacy `config/quickshell/axiom-shell` path: it is documented as deprecated and is not the active runtime config.
- Kept Axiom host-specific facts generated by Nix under Hyprland override files such as `monitors.conf`, `workspaces.conf`, and `hypr/custom/*.conf`, including `$qsConfig = ii` and `$dontLoadDefaultExecs = 1`.
- Added local integration adjustments recorded in the manifest, including a Quickshell `0.3.0` build for `Quickshell.Services.Polkit`, a Nix-provided Python environment for helper scripts, and default disabling of the external KDE polkit agent to avoid duplicate agents.

## Security And Privacy Notes

- Upstream `setup`, installer state, generated color outputs, API keys, credentials, live cache/state, local user config JSON, and generated wallpaper/color state were intentionally omitted per `docs/import-manifest.md`.
- The recorded secret scan found no committed keys or credentials matching the checked common patterns.
- Imported `Quickshell.Services.Polkit` provides authentication UI and, per review, does not bypass authentication. The external KDE polkit agent is disabled by default to avoid duplicate agents.
- AI/cloud/API source is present because it is part of the imported end4 source surface, but review records that online calls require user-provided keys and command-execution tooling has an explicit approval path.
- `cliphist` remains privacy-sensitive because text/image capture is default-on through a Nix-owned watcher; retention/clear policy remains follow-up.
- Imported live-session scripts for wallpaper, screenshot/OCR, recorder, Hyprland, keyring, and AI helpers were not run during review.

## Residual Live-Session Risks

- `systemctl --user restart quickshell.service` was not run from this TTY environment.
- Actual Axiom graphical session still needs exercise of sidebars, overview/search, notifications, OSD, lock/session, wallpaper switching, polkit prompts, tray behavior, and hardware-backed controls for audio, brightness/DDC, network, Bluetooth, and power profiles.
- Some upstream optional actions still assume KDE tools, `kitty`/`fish`, or Arch-style update commands; follow-up should map `Config.options.apps.*` to Axiom/Nix defaults or hide unsupported actions.
- AI/cloud/function tooling remains powerful and should receive follow-up policy/documentation hardening before broader enablement.
- `.upstream/` was noted by review as a local import clone that should remain uncommitted or be cleaned before PR finalization.
