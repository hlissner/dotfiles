# RFC: Complete Axiom End4 Desktop Import

> Status: design for no-downgrade implementation, pending review-rfc
> Type: RFC Standard, implementation-bound
> Date: 2026-05-09
> Task: `.legion/tasks/dots-hyprland-desktop-rfc/`

## Executive Summary

The prior Phase 4 PR shipped a service substrate and explicitly left `config/quickshell/ii/shell.qml` as prior-phase debt. The user rejected that downgrade and pointed to `https://github.com/end-4/dots-hyprland` as the missing source. This RFC supersedes the substrate-only decision.

Decision: import the required end4 desktop source into this repository and make Axiom's active Quickshell config load `ii/shell.qml`. Nix remains the owner of deployment, host facts, services, permissions, generated-state boundaries, UWSM/greetd/portal startup, and rollback knobs. Upstream installer behavior, secrets, generated cache/state, and live home-directory mutation are not imported.

## Goals

- Vendor the upstream end4 `ii` Quickshell shell family so `quickshell --config ii` resolves `ii/shell.qml`.
- Default Axiom Quickshell to `ii`, not `axiom-shell`.
- Import the adjacent end4 config surfaces the shell expects: matugen templates, colors scripts, Hyprland source layering, hypridle/hyprlock configs, and custom override slots.
- Keep Axiom host facts in Nix: monitors, workspaces, default apps, UWSM/greetd, portals, services, groups, keyring/polkit, cliphist, power profiles, and i2c/DDC.
- Keep end4 theme direction authoritative: `IllogicalImpulseFamily`, Material Symbols / Google Sans-style fonts, Material/matugen/wallpaper runtime chain, no `autumnal` desktop visual truth.
- Record validation that distinguishes host identity (`axiom`) from live Hyprland session availability.

## Non-Goals

- Do not run upstream `setup`.
- Do not import mutable installer state, package-manager scripts, generated color outputs, API keys, cloud credentials, or live cache/state.
- Do not preserve old Axiom dock/guide/buttons as compatibility requirements.
- Do not fully enable Phase 5 visual automation or Phase 6 AI policy. Source files may be present if upstream QML imports require them, but network/API behavior must remain inert without explicit policy and keys.

## Baseline

- Axiom is currently on PR #18: Quickshell service substrate exists, but the active shell source is still historical `config/quickshell/axiom-shell` unless changed.
- `config/quickshell/ii/**` is absent on `origin/master` before this task.
- The current command host is `axiom`, but the tool shell can be `tty` without `WAYLAND_DISPLAY` or `HYPRLAND_INSTANCE_SIGNATURE`; live shell restart validation depends on session variables, not hostname.

## Alternatives

### A: Keep substrate-only and document prior-phase debt

- Pros: smallest diff; already buildable.
- Cons: explicitly rejected by the user; keeps old shell as de facto UX; fails `ii/shell.qml` acceptance.
- Decision: rejected.

### B: Run upstream end4 setup on Axiom

- Pros: closest to upstream behavior quickly.
- Cons: violates Nix ownership, mutates live home config, bypasses review/rollback, can import secrets/state.
- Decision: rejected.

### C: Vendor required upstream source, wire with Nix

- Pros: satisfies no-downgrade requirement, keeps repository reviewability and Nix ownership, allows future pruning.
- Cons: large import and broad runtime surface; may need follow-up package/runtime fixes.
- Decision: chosen.

## Design

### Source Import Boundary

Import from upstream `dots/.config` into repository `config/` paths using this reviewable manifest rule:

- `dots/.config/quickshell/ii` -> `config/quickshell/ii`
- `dots/.config/matugen` -> `config/matugen`
- `dots/.config/hypr` -> `config/hypr`, with Axiom overrides generated by Nix under `custom/`
- `dots/.config/fuzzel` -> `config/fuzzel` when needed for end4 fallback launcher/theme outputs
- any path under `dots/.config/quickshell/ii/**` is included as part of the shell, including `scripts/**`, service QML, modules, assets, panel families, and helper files
- any non-`ii` upstream path may only be imported if a copied `ii` file or copied Hyprland/matugen file references it; the PR must list that path and reason in `docs/import-manifest.md`

Do not import upstream `setup`, `sdata/**`, distro package manager scripts, generated state/cache, API keys, credentials, local user config JSON, or files whose only purpose is mutating live home outside repository/Nix ownership. The PR must create `docs/import-manifest.md` with upstream commit, copied path list, omitted path list, and omission reasons.

### Quickshell Ownership

`modules/desktop/quickshell.nix` should default `configName = "ii"` and link `quickshell/ii`. The Quickshell user service should continue under `hyprland-session.target` and restart through existing hooks.

The old `axiom-shell` path should not be active. It may be removed or left as non-linked historical source only if keeping it is necessary for review continuity; it must not be the default config.

### Hyprland Ownership

Use end4 source layering as the repository base. Nix should generate Axiom-specific override files instead of rewriting upstream core files:

- monitor facts -> `hypr/monitors.conf`
- workspace facts -> `hypr/workspaces.conf`
- Axiom env/default apps -> `hypr/custom/env.conf` and/or `hypr/custom/variables.conf`
- local rules/keybind fallbacks -> `hypr/custom/rules.conf`, `hypr/custom/keybinds.conf`

UWSM/greetd/portal/session-target startup remains owned by NixOS. If upstream `exec-once` conflicts with Nix service ownership, prefer Nix ownership and document the override.

### Theme And Generated State

Matugen templates and scripts are repository-managed. Generated color files remain mutable runtime outputs under XDG state/config/cache targets, not committed sources or Nix store outputs.

`autumnal` is not desktop truth. Any remaining autumnal module must not write Hyprland desktop visuals.

### Phase 4 Runtime

The Phase 4 service substrate from PR #18 remains required and can be extended to satisfy upstream QML imports. This includes Qt/Kirigami/QML modules, Material Symbols / Google Sans-style fonts, keyring, polkit agent, cliphist, power profiles, DDC/i2c, network/Bluetooth/audio/media/resource/fallback tools.

### Security And Privacy

- Clipboard history remains privacy-sensitive; `cliphist` retention pruning is follow-up unless needed for immediate loadability.
- AI/cloud/API modules must remain inert without user-provided keys and policy.
- Polkit agent provides UI prompts but must not bypass authentication.
- Imported upstream source must be reviewed for accidental secrets or generated state before commit.

### Hypridle And Hyprlock

If upstream stores `hypridle.conf`, `hyprlock.conf`, or `hyprlock/**` under `dots/.config/hypr`, they are included through the `config/hypr` import. If upstream stores them as sibling `.config` paths, import them into the matching repository `config/` path and record them in `docs/import-manifest.md`.

## Implementation Plan

- Fetch upstream in the worktree and record upstream commit.
- Copy required source directories into `config/`.
- Write `docs/import-manifest.md` with copied and intentionally omitted upstream paths.
- Change Quickshell default to `ii` and adjust Nix links/packages.
- Replace or merge Hyprland config with end4 layering while regenerating Axiom host overrides.
- Remove/demote old Axiom shell runtime and old guide remnants.
- Update task evidence and wiki.

## Verification Strategy

- Confirm `config/quickshell/ii/shell.qml` exists and references `IllogicalImpulseFamily`.
- Run a static import scan over copied `ii` QML files to catch obvious missing relative source paths.
- Run the strongest available Quickshell/QML loadability command in the current environment. Preferred order: a real user-session `systemctl --user restart quickshell.service`; otherwise a bounded direct `quickshell --config ii` smoke attempt with captured stderr; otherwise available Qt/QML tooling such as `qmllint` with explicit import paths. A headless failure is only acceptable if the report shows the command reached a compositor/session limitation rather than a missing `ii` file, missing QML import, or missing package.
- Run targeted `nix eval` for Quickshell service ExecStart, linked config paths, generated Hyprland overrides, required services, and package presence.
- Run `nix build --impure .#nixosConfigurations.axiom.config.system.build.toplevel --no-link`.
- Run static scans to ensure upstream setup/generated state/secrets are not committed.
- If possible, run `systemctl --user restart quickshell.service` and inspect `journalctl --user -u quickshell.service -b` only when a live Axiom user/Wayland session is actually available.
- If possible, run evaluated Hyprland config verification; if upstream syntax or live session blocks it, record the exact limitation.

## Rollback

- Merge-time rollback: revert the PR.
- Runtime rollback: switch to previous NixOS generation.
- If Quickshell `ii` fails: temporarily override `modules.desktop.quickshell.configName = "axiom-shell"` locally only as an emergency rollback, not as product truth.
- If Hyprland layering fails: revert to previous generation or restore previous config PR.

## Decision

Proceed with option C: vendor required upstream end4 source and wire it through Nix. Substrate-only is invalid for this continuation.
